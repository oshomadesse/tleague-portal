# Tリーグポータル - 仕様書

3人麻雀のスコア記録（友達と同期）と、待ち牌・翻数チェッカー（個人ローカル）を1つのHTMLに統合したWebアプリ。

## 公開先

- 本番URL: https://oshomadesse.github.io/tleague-portal/
- リポジトリ: https://github.com/oshomadesse/tleague-portal
- ホスティング: GitHub Pages（main / root）

## 構成

- 形式: 単一HTMLファイル（CSS・JS内包）
- 上部タブでスコアシート / チェッカーを切替（デフォルト: スコアシート）
- パスワード認証はポータル全体の入口（認証通れば両方アクセス可）

## ファイル

```
🀄 mahjong-portal/
├── index.html       # メインアプリ
├── favicon.ico

```

## タブ1: スコアシート

3人打ち麻雀のスコアを記録・管理。**API経由で友達と同期**。

### プレイヤー（固定値）

| プレイヤー | 表示名 |
|---|---|
| 1P | 🛡️ジェッツ西方🛡️ |
| 2P | 🐺ビースト高橋🐺 |
| 3P | 🐲ファイト大田🐲 |

### 機能
- スコア入力（点数、役満チェック、自動ポイント計算）
- ランキング表示（今日 / 通算）
- 対局履歴（節タブ切替、編集・削除）
- 節管理（新規作成、削除）
- API経由データ同期 + ローカルストレージ

### API

- エンドポイント: `https://mahjong-scoresheet-api.oshomadesse.workers.dev/mahjong`
- 認証: `x-access-token` ヘッダー（パスワード）、セッション2時間
- バックエンド: Cloudflare Workers
- ※パスワードはWorkers側の環境変数で管理。リポジトリには絶対に入れない

### ポイント計算

- 基礎: `(点数 - 40000) / 1000`
- 3着ペナルティ: 通常-10、2位3位同点なら各-5
- 1位は自動調整（合計0）
- 飛び -10 / 飛ばし +10 / ダブル飛ばし +20
- 役満 +40（他-20）/ ダブル役満 +80（他-40）

## タブ2: チェッカー

サンマ（三人麻雀）の手牌から待ち牌を判定し、翻数・役構成を自動計算。**完全ローカル動作（API・ストレージ通信なし）**。

### ルール
- マンズは1・9のみ、北は抜きドラ扱い、連風牌・三色同順は不採用

### 機能
- 鳴き入力（なし／1〜4鳴き、ポン／暗槓／明槓、種類選択）
- 牌選択（マンズ1/9、ピンズ・ソーズ各1〜9、字牌7種、最大4枚）
- 待ち牌判定（通常手 / 七対子 / 国士無双 / 13面待ち）
- ヒアリング（和了種別、立直、状況、場風、自風、赤、北抜き、ドラ、裏ドラ）
- 翻数計算と役構成表示
- 矛盾組み合わせの自動グレーアウト

### 同期しない理由

友達と同期されると、上がり牌の検討内容が見られてしまう。チェッカーは個人の検討用なので、APIもlocalStorageも使わずに完全ローカルで動作させる。

## 開発メモ

### CSS命名の注意
スコアシートとチェッカーは元々別アプリだったため、CSSクラスや関数名の衝突がない構造になっている（マージ時に確認済み）。新規追加時に `.container` `.btn` `.section` 等の汎用名を当てると衝突するので注意。

### checker初期化の保険
`window.__checkerInit` でグローバル化し、タブ切替時にも再実行する保険を入れている。HTMLパース順序やタイミング問題で初期化が走らなかった場合の救済策。

### HTML構造のリスク
スコアシート原本にあった `<div class="input-grid">` の閉じタグ漏れが、portal化したときにラッパー div を巻き込む形でレイアウトを壊した（チェッカータブが scoresheet 内に入ってしまった）。HTML構造の整合性は重要。

### 認証フロー
- 初期: passwordOverlay が画面全体（z-index:10000）を覆う
- portal-tabs-bar は z-index:100 → overlay の下、未認証では触れない
- パスワード入力成功 → overlay非表示 → タブ操作可能

### 公開URL
GitHub Pages の URL は `https://oshomadesse.github.io/tleague-portal/`。Cloudflare Workers のCORS許可は同一オリジン `https://oshomadesse.github.io` で動作。
